一、核心规则
1、确认使用中文注释，使用中文编写报错日志等
2、确认注释代码已封装为函数
3、确认所有变量都有注释
4、合并重复逻辑，避免冗余，使用函数或模块来封装重复逻辑，提高代码复用性
5、确认使用ES6+语法
6、调试函数提取：将所有console.log、alert等调试代码提取到独立的函数中，方便管理和关闭。
7、函数参数优化
8、确认使用封装好的工具函数，如：handleApiCall、showErrorFn等

二、文件结构
按照以下区域顺序组织代码：
// ===== 1. 依赖导入区域 =====
// ===== 2. TODO待处理导入区域 =====
// ===== 3. 状态控制逻辑区域 =====
// ===== 4. 通用工具函数区域 =====
// ===== 5. 注释代码函数区 =====
// ===== 6. 错误处理函数区域 =====
// ===== 7. 数据处理函数区域 =====
// ===== 8. UI渲染逻辑区域 =====
// ===== 9. 页面初始化与事件绑定 =====
// ===== 10. TODO任务管理区域 =====
// ===== 11. 导出区域 =====

三、命名规范
1、变量命名：使用驼峰命名法snakeCase，布尔值（is、has、can前缀），函数（get、set、handle、show前缀）
2、注释函数命名：使用驼峰命名法snakeCase，描述性强，使用前缀区分（如：handleUserAuth、showDataCache）
3、注释代码函数处理规则：
- **返回值**：保持原有结构
- **多层嵌套**：整体封装，保持内部结构
- **函数调用**：仅作代码保存容器，不实际执行

四、校验检查
1、buildbuild和lint：确保代码符合规范，无错误和警告
2、是否有未使用的变量或函数
3、是否有重复的注释函数
4、是否有未封装的注释代码

五、AI执行优先级
1、核心规则（最高）
2、必需函数：errorShowFn、handleApiCall、工具函数、注释函数区函数
3、模式应用：API调用模式、11区域结构、命名约定
